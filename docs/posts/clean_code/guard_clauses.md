# Guard Clauses

중첩된 if-else if-else 문들은 depth가 깊어질수록 callback hell처럼 가독성을 저하시킬 수 있습니다.

```js
if (조건 1) {
	if (조건 2) {
		if (조건 3) {
			// ...
		} else {

		}
	} else {

	}
} else {

}
```

일련의 조건들을 직렬적으로 만족여부를 확인하는 것보다는 조건을 만족하지 않은 경우에는 중간에 return하고 만족하는 경우 return 아래 로직을 사용하도록 적는 것이 훨씬 더 이해하기 쉬워집니다.

```js
if (!조건 1) {
	// ...
	return;
}
if (!조건 2) {
	// ...
}
if (!조건 3) {
	// ...
}

// 이쪽 영역에 조건 1,2,3을 모두 만족하는 경우에 수행할 로직이 위치한다.

```
